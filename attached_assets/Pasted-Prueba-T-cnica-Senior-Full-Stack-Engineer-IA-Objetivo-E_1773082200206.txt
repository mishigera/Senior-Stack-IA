Prueba Técnica – Senior Full-Stack Engineer (IA)
Objetivo
Evaluar las habilidades técnicas, de diseño y de resolución de problemas de candidatos para la vacante de
Senior Full-Stack Engineer (IA), alineado con los requerimientos del área de Desarrollo en Toka.
Esta prueba valida conocimientos en desarrollo Full-Stack, arquitectura de microservicios, integración de
agentes de IA y buenas prácticas de ingeniería de software en un entorno local dockerizado.

Alcance esperado
Esta prueba técnica se divide en dos partes:
Primera parte: Los ejercicios 1, 2, 3 y 4 conforman la primera parte de la prueba, enfocada en el desarrollo
completo de un sistema de gestión de usuarios con microservicios, front-end y resolución de problemas.
Segunda parte: El ejercicio 5 (Integración de IA y Agentes) constituye la segunda parte de la prueba,
enfocada en la integración de agentes de IA con el sistema desarrollado en la primera parte.

1. Diseño y Arquitectura de Software
Objetivo: Validar conocimientos en patrones de diseño, arquitecturas modernas, Domain-Driven Design
(DDD), Clean Architecture y buenas prácticas de diseño orientado a objetos en un entorno de
microservicios.
Instrucción:
Diseña la arquitectura para un sistema de gestión de usuarios que permita autenticación, autorización,
auditoría de accesos y gestión de roles. El sistema debe ser escalable y estar basado en microservicios,
con capacidad de ejecutarse en un ambiente local dockerizado.
Requisitos técnicos:
Microservicios: Diseñar al menos 3-4 microservicios independientes (ej: Auth Service, User Service,
Audit Service, Role Service)
Comunicación: Incluir comunicación síncrona (REST) y asíncrona (RabbitMQ o Kafka) entre
microservicios
Estrategia de datos: Diseñar estrategia multi-DB considerando SQL Server/MySQL/Postgres (datos
transaccionales), MongoDB (auditorias), Redis (caching)
Integración de IA: Incluir en la arquitectura un microservicio o módulo para integración de agentes
de IA (OpenAI/Azure OpenAI) con capacidad de RAG (Retrieval-Augmented Generation)
Ambiente dockerizado: Diseñar arquitectura ejecutable localmente con Docker Compose,
incluyendo todos los servicios necesarios

2. Implementación de Microservicios
Objetivo: Comprobar habilidades prácticas en desarrollo de microservicios, implementación de
arquitecturas distribuidas, comunicación entre servicios y aplicación de principios DDD y Clean
Architecture.
Instrucción:
Implementa los microservicios diseñados en el ejercicio 1. Debes desarrollar al menos 3-4 microservicios
del sistema de gestión de usuarios (ej: Auth Service, User Service, Audit Service, Role Service). Cada
microservicio debe ser independiente, dockerizado y seguir los principios de la arquitectura diseñada.
Requisitos técnicos:
Microservicios: Implementar al menos 3-4 microservicios independientes del sistema diseñado
Comunicación: Implementar comunicación síncrona (REST) entre microservicios y al menos un
ejemplo de comunicación asíncrona (RabbitMQ o Kafka) para eventos
Arquitectura: Aplicar principios de DDD y Clean Architecture en cada microservicio
Autenticación: Implementar autenticación OAuth2/OIDC distribuida entre microservicios (puede usar
JWT con validación distribuida)
Base de datos: Usar estrategia multi-DB según diseño (SQL Server, MySql, Postgres, MongoDB,
Redis según corresponda)
Dockerización: Cada microservicio debe tener su Dockerfile y un docker-compose.yml que
orqueste todos los microservicios y dependencias (DBs, Redis, RabbitMQ/Kafka, etc.)
Tests: Incluir tests unitarios con coverage mínimo del 70%
Logging: Implementar logging estructurado (JSON) en cada microservicio
Tecnologías requeridas:
Lenguaje libre: C# (.NET 8+), Python (FastAPI) o Node.js (Express/NestJS)

5. Integración de IA y Agentes

Objetivo: Validar habilidades en integración de agentes de IA, implementación de RAG (Retrieval-
Augmented Generation), prompt engineering y evaluación de respuestas de LLMs.

Instrucción:
Integra un agente de IA en el sistema de microservicios desarrollado en el ejercicio 2. El agente debe ser
implementado como un microservicio adicional o como parte de un microservicio existente, y debe
integrarse con los microservicios ya desarrollados.
Requisitos técnicos:
Agente de IA: Integrar un agente usando OpenAI, Azure OpenAI, Anthropic o Gemini APIs
RAG básico: Implementar RAG (Retrieval-Augmented Generation) con vector database
Opciones: Pinecone, Weaviate, Azure Cognitive Search, o usar un contenedor local (Qdrant,
Chroma, etc.)
Crear pipeline de embeddings para documentos/contexto
Prompt engineering: Diseñar y documentar estrategia de prompt engineering (system prompts,
few-shot examples, chain-of-thought si aplica)
Evaluación: Implementar evaluación básica de respuestas:
Métricas de latencia (tiempo de respuesta)
Coste de tokens (input/output)
Validación básica de calidad de respuesta
Integración: El agente debe integrarse con al menos uno de los microservicios del ejercicio 2 (ej:
responder consultas sobre usuarios, generar reportes, etc.)
Tecnologías sugeridas:
APIs de IA: OpenAI, Azure OpenAI, Anthropic, Gemini
